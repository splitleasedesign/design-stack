# Listing Creation Flows by Host Archetype

> **Date:** October 31, 2025
> **Analyst:** Aristotle (Anthropologist & UX Researcher)
> **Purpose:** Functional flows optimized for each host archetype's psychological needs

---

## Overview

Each host archetype has fundamentally different needs, anxieties, and decision-making styles. A single flow cannot serve all equally. Below are optimized flows for each archetype, plus a "standard" flow that attempts to serve the middle ground.

**Key Insight:** The flow isn't just about collecting informationâ€”it's about **anxiety management** and **building trust** at each step.

## Key Design Insights

### 1. **One Size Does NOT Fit All**
The flows must diverge based on archetype. Forcing Cristina through Dimitri's flow will cause abandonment. Forcing Dimitri through Anthony's flow will trigger distrust.

### 2. **Time Expectations Matter**
Always state upfront: "This takes X minutes." Americans hate uncertainty about time commitment.

### 3. **Save & Exit = Anxiety Relief**
Verifiers need this. It signals "I can leave and think about it" which paradoxically makes them more likely to complete.

### 4. **Agent Option = Safety Net**
For Delegators, "we'll call you" is the product. For Verifiers, it's a fallback. Always visible.

### 5. **Progressive Disclosure > Long Forms**
Break complex flows into steps. Never show all fields at once (cognitive overload).

### 6. **Default to Speed, Expand for Control**
Standard flow should start simple with expansion points. Easier to add detail than remove it.

### 7. **Trust Signals at High-Anxiety Moments**
Right before "Publish," remind them of protections. Before pricing, show data. Before photos, show incentive.

## Recommended Implementation Strategy

### Phase 1: MVP
Build **Quick Path only** (like Passive/Opportunist flow)
- Serves Anthony, Cristina, Robert types
- Fast to build
- Agent-assisted to complete details
- 80% of hosts may be satisfied with this

### Phase 2: Add Control Layer
Add **Complete Path option** (like Verifier flow)
- Serves Dimitri, Stephanie, Tag types
- Bifurcated onboarding (choose your path)
- Reduces agent burden for self-sufficient hosts

### Phase 3: Adaptive Intelligence
Build **Standard Flow with adaptive elements**
- Platform learns from behavior (if user edits recommended price, offer more customization)
- Dynamic path switching (start quick, offer "want more control?")
- Personalized based on host signals
