- **Phase definitions**:
  - discovery: Host first hears about Split Lease (referral, ad, search). Deciding whether to explore.
  - evaluation: Host researches the platform. Compares to Craigslist, Airbnb, roommate apps. Reads about guarantees.
  - onboarding: Host signs up, verifies identity, connects with their assigned agent (e.g., Bryant). First call happens here.
  - listing_creation: Host creates their listing: photos, description, amenities, house rules, availability calendar.
  - pricing: Host sets nightly/weekly rates. Understands the payment model, guaranteed income, Split Lease fee structure.
  - proposal_mgmt: Host receives guest proposals, reviews them, counters terms, accepts or declines. Upon acceptance, Split Lease sends lease documents to both parties.
  - active_lease: Guest is living in the space. Host accesses the Leases page to track payments and manage stays. Both parties use the Date Change Tool. Guest notifies Host of arrival and sends cleaning photos. Both parties review each stay.
  - retention: Lease ends or renews. Host decides whether to list again, refer others, or leave the platform.

- Labels are a last resort: Present data without labels when format/context is sufficient. Combine labels and values into single units.

- Typography: Establish type scales. Keep line length in check. Line-height is proportional.
- Working with Color: Define shade palettes up front. Use HSL. Don't rely on color alone.
